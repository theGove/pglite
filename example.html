Shared PGlite multi-iframe demo

Both iframes load the studio with ?shared=1 so they share one idb://websql-studio database via PGliteWorker. Without that parameter, each visit gets a private in-memory DB. Serve this folder over HTTP (not file://), then:

  1. In Frame A, click Run to create demo_shared and insert a row.
  2. In Frame B, click Run (or refresh tables) — you should see the same table/data.
  3. Optional: Clear Data in one frame; after the other frame reloads or clears, both see an empty DB.
